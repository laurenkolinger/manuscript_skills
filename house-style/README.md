# house-style

**Purpose.** Fix the writing mechanics of every piece of prose so it reads as if an accountable person did real work and stated it plainly.

## When to use

Apply it to all written deliverables, even short ones, and even when another
skill is already active: emails, summaries, proposals, slide text, technical
narratives, analyses, reports, figure captions, READMEs, session logs, and
manuscript or SI prose. This is the foundational layer, so default to running it
whenever you produce prose.

## What it does

It enforces six mechanics and leaves strategy alone:

- No em dashes. Use periods, commas, colons, semicolons, or parentheses.
- State claims affirmatively. Lead with what is true; drop "not X, but Y."
- Match tense to commitment. Past for completed work, present for current
  capacity, future for planned work. Methods and Results stay in the past tense.
- Put a real actor in the subject (we, the team, named partners), and let tools
  and workflows be what those actors use.
- Deliver the finished artifact. Strip planning talk and "the user wants" residue.
- Write sentences that stand alone.

American spelling throughout. The skill ends with a quick pre-send checklist.

## Fit in the workflow

house-style is the base layer of the writing stack. `analytical-writing` adds
intellectual honesty, the scientific-writing suite (`scientific-results-writing`,
`de-densify-scientific-prose`, `scientific-sentence-framing`) shapes results
prose, `humanizer` runs as the legibility pass, and `reporting` governs figures
and numbers. All of them sit on top of house-style. In the manuscript family it
runs during authoring in the base layer and again during the edit layer's
interactive review, so coauthor-driven revisions meet the same bar as the
original draft. The base draft is v0 until it enters the edit template.
